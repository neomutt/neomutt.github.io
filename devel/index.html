What's Going On?

Latest Development

There's a new version of NeoMutt based on the newly released Mutt-1.7.0. Although not ready for the general public, developers are asked to give it a try and find all the bugs.

Development

Any development on features is done in "devel/" feature branches in the NeoMutt repository. Once development is complete, the work will be moved into a "feature/" branch.

There are more development pages:

  • GitHub - Descriptions of all the NeoMutt repositories and how the branches fit together
  • Coding Style - The preferred style for code committed to NeoMutt
  • Tasks - A list of tasks to be done
  • Obsolescence - A list of old projects that can be forgotten

Finally, some automatic checking is done on commits and pull-requests for NeoMutt. It benefits from two free services (free for Open Source). They are triggered on every commit to GitHub.

Travis CI automatically build projects on every commit (Continuous Integration). Coverity performs static analysis on the code looking for potential problems.
